- React 18+ SPA (Vite)

Backend:
- Express.js server running on 0.0.0.0:3000

- Supports guest + authenticated modes

Database:
- MongoDB (Mongoose optional)
- Must support fallback to in-memory + localStorage

## 3. MULTI-MODEL ROUTING LOGIC

The system uses a fallback inference chain:

1. If GITHUB_TOKEN exists:
   → Use GitHub Models API (primary inference)

2. Else if GEMINI_API_KEY exists:
   → Use Gemini (fallback inference)

3. Else:
   → Use Local Shard Simulator
   → Must return structured, helpful responses (never fail silently)

---

## 4. USER STATE SYSTEM

Two states exist:

### Guest Mode
- Anonymous session: guest_XXXX@computeinfra.network
- Stored in localStorage
- Full chat access via model routing pipeline
- Data is transient

### Authenticated Mode
- Persistent identity via email
- Full system access
- Permanent chat storage and history
